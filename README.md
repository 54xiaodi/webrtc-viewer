# RDK X5 WebRTC Viewer — Flutter Cross-Platform App

## Prerequisites

- Flutter SDK 3.2+ ([install guide](https://docs.flutter.dev/get-started/install))
- Android Studio (for Android builds + emulator)
- Codemagic account (for iOS builds without a Mac)

---

## Step 1: Initialize Flutter Project

The Dart code and assets are ready. You need to generate Flutter's platform scaffolding:

```bash
cd flutter_app

# Generate the full Flutter project structure
flutter create --project-name rdk_webrtc_viewer --org com.rdkx5 .

# Get dependencies
flutter pub get
```

This generates `android/`, `ios/`, `web/`, `test/` directories with all platform boilerplate.
Your `lib/main.dart`, `pubspec.yaml`, and `assets/` are preserved.

---

## Step 2: Configure Android

After `flutter create .`, edit the generated `android/app/src/main/AndroidManifest.xml`:

```xml
<manifest ...>
    <!-- Add permissions -->
    <uses-permission android:name="android.permission.INTERNET" />
    <uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />

    <application
        ...
        android:usesCleartextTraffic="true">  <!-- Allow ws:// connections -->

        <activity
            ...
            android:screenOrientation="landscape"
            android:keepScreenOn="true"
            android:configChanges="orientation|screenSize|keyboardHidden">
```

Set the minimum SDK version in `android/app/build.gradle`:
```groovy
android {
    defaultConfig {
        minSdk = 24  // Change from 21 to 24 (required for WebView WebRTC)
    }
}
```

---

## Step 3: Configure iOS

After `flutter create .`, edit `ios/Runner/Info.plist`:

```xml
<!-- Add inside <dict> — allow cleartext ws:// connections -->
<key>NSAppTransportSecurity</key>
<dict>
    <key>NSAllowsArbitraryLoads</key>
    <true/>
</dict>

<!-- Force landscape orientation — replace existing orientation keys -->
<key>UISupportedInterfaceOrientations</key>
<array>
    <string>UIInterfaceOrientationLandscapeLeft</string>
    <string>UIInterfaceOrientationLandscapeRight</string>
</array>
<key>UISupportedInterfaceOrientations~ipad</key>
<array>
    <string>UIInterfaceOrientationLandscapeLeft</string>
    <string>UIInterfaceOrientationLandscapeRight</string>
</array>
```

Set iOS deployment target in `ios/Podfile` (uncomment and set):
```ruby
platform :ios, '14.0'
```

---

## Step 4: Build & Run on Android (Local)

```bash
# Connect Android device via USB
flutter devices          # Verify device is listed
flutter run              # Build and install debug version
flutter build apk        # Build release APK
```

The APK is at: `build/app/outputs/flutter-apk/app-release.apk`

## Step 5: Build iOS via Codemagic (Cloud Mac)

Since you don't have a Mac, use Codemagic's cloud Mac service:

### 5.1 Push to Git

```bash
cd flutter_app
git init
git add .
git commit -m "Initial Flutter WebRTC viewer"
git remote add origin https://github.com/YOUR_USER/rdk-webrtc-viewer.git
git push -u origin main
```

### 5.2 Set Up Codemagic

1. Go to [codemagic.io](https://codemagic.io) and sign in with GitHub
2. Click **Add application** → select your repository
3. Choose **Flutter App** as the project type
4. Codemagic detects `codemagic.yaml` automatically

### 5.3 Configure iOS Code Signing (Required)

In Codemagic dashboard → your app → **Settings**:

**Option A: Development/Ad-Hoc (no App Store)**
1. Go to **Code signing identities**
2. Upload your Apple Development certificate (.p12) and provisioning profile
3. Or use **Automatic code signing** with your Apple Developer account

**Option B: No Apple Developer Account (simulator only)**
- Skip code signing — build for simulator:
  ```yaml
  # In codemagic.yaml, change the build script to:
  flutter build ios --debug --no-codesign --simulator
  ```

### 5.4 Trigger Build

1. Push a commit to trigger automatic build
2. Or click **Start new build** in Codemagic dashboard
3. Select **ios-build** workflow
4. Wait ~10 minutes for the build
5. Download the IPA from **Artifacts**

### 5.5 Install IPA on iPhone

- **With TestFlight:** Upload IPA to App Store Connect → distribute via TestFlight
- **With ad-hoc provisioning:** Install via Apple Configurator or Xcode
- **With AltStore:** Sideload the IPA without App Store

## How It Works

The app is a thin Flutter wrapper around a WebView:

1. **`main.dart`** renders a fullscreen `WebViewWidget` with native overlay controls
2. **`webrtc_viewer.html`** (loaded from assets) handles all WebRTC logic:
   - WebSocket signaling to RDK X5 on port 8080
   - SDP offer/answer exchange
   - ICE candidate exchange
   - H.264 video decoding and rendering
3. **Communication bridge:**
   - Flutter → JS: `_controller.runJavaScript("connectTo('192.168.3.105')")`
   - JS → Flutter: `console.log("STATUS:Streaming")` → `JavaScriptChannel` → status bar update

This approach uses the platform's built-in WebRTC engine (Chrome on Android, WebKit on iOS)
instead of a native WebRTC SDK, avoiding hardware decoder compatibility issues.
